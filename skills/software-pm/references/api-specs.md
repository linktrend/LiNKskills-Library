# API & Tool Technical Specifications

## Inputs
- PRD or specification path.
- Factory alignment (`saas` or `web`) from root `factory.json`.
- Delivery window and known constraints.

## Outputs
- Technical Backlog (epics/stories/tasks) with dependency ordering.
- Definition of Done matrix including QA sign-off requirement.

## Governance
- Every backlog item must map to PRD intent and factory module.
- Done criteria must include QA completion signal.
