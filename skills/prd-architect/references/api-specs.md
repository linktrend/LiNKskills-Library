# API & Tool Technical Specifications

## Discovery Inputs
- Idea statement or vibe description.
- Business objective and target user.
- Constraints, assumptions, and existing assets.

## Expected Deliverable
- Professional technical specification containing:
  - Problem statement and goals/non-goals
  - Architecture/module breakdown
  - Data contracts and acceptance criteria
  - Delivery phases and risk log
- Mandatory format: `references/MASTER_PRD_TEMPLATE.md` for every new project briefing.

## Supporting Interfaces
- `read_file` for local context and historical specs.
- `write_file` for staged spec output and checkpoints.
